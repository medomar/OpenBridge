### Quick Start

```bash
cd demos/09-cafe-restaurant/app
npm install
npm start
```

Then open:

- **Customer form:** http://localhost:3200
- **Admin dashboard:** http://localhost:3200/admin.html

### How It Works

1. **Admin** opens the admin dashboard and sets the daily menu (1 entrée + 3 plats) for a specific date
2. **Customer** opens the reservation form, picks a date, and sees the day's menu
3. Customer selects a formula (Entrée + Plat, Entrée only, or Plat only)
4. If the formula includes a plat, customer assigns quantities per dish (up to number of guests)
5. Customer submits — reservation appears instantly on the admin dashboard

### API Endpoints

| Method | Endpoint            | Description                     |
| ------ | ------------------- | ------------------------------- |
| `POST` | `/api/dishes`       | Set dishes for a date (admin)   |
| `GET`  | `/api/dishes`       | List all configured dishes      |
| `GET`  | `/api/dishes/:date` | Get dishes for a specific date  |
| `POST` | `/api/reservations` | Create a reservation (customer) |
| `GET`  | `/api/reservations` | List all reservations (admin)   |

## Common Questions

**Q: Is the data persistent?**
A: This demo uses in-memory storage. For production, connect a database (SQLite, PostgreSQL, etc.).

**Q: Can I customize the menu options?**
A: Yes — the admin dashboard lets you set different dishes for each date.

**Q: Can multiple staff manage reservations?**
A: Yes. The admin dashboard is accessible to anyone with the URL. Add auth for production use.

**Q: What if I need more time slots?**
A: Edit the `<select>` options in `public/index.html` or make them configurable via the admin.
