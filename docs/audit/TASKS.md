## Phase 41: Memory & Startup Fixes

## Phase 42: Exploration Pipeline Fixes

> Fix the exploration system so results are actually persisted and readable. The current pipeline silently loses all exploration data because: (a) ExplorationCoordinator has no JSON fallback for workspace-map after OB-810 removed it, (b) `readWorkspaceMapFromStore()` only checks memory and ignores the JSON file on disk, (c) `storeExplorationChunks()` silently swallows all failures, and (d) `loadExplorationSummary()` marks empty maps as "completed" preventing re-exploration.
